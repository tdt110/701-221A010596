Phần 1: Lý thuyết (40 điểm)

Câu 1 (10đ): position: absolute vs position: relative

position: relative giữ phần tử ở vị trí bình thường trong luồng tài liệu (normal flow), nhưng cho phép dịch chuyển nó (qua top/left/right/bottom) so với chính vị trí gốc của nó mà không ảnh hưởng đến các phần tử xung quanh. Nó cũng thường được dùng làm "điểm neo" (containing block) cho các phần tử con có position: absolute.

position: absolute kéo phần tử ra khỏi luồng tài liệu hoàn toàn, định vị nó dựa trên phần tử cha gần nhất có position khác static (thường là relative). Nếu không có cha nào như vậy, nó định vị theo <body>.

Ví dụ tình huống responsive: Một card sản phẩm dùng position: relative để làm khung chứa, bên trong có một nhãn "Giảm giá" dùng position: absolute; top:0; right:0; để luôn nằm ở góc trên-phải bất kể card co giãn theo kích thước màn hình. Cách này giúp layout responsive vì nhãn luôn bám theo card cha thay vì theo toàn trang.

Ảnh hưởng đến deploy/hosting: Việc dùng sai absolute (ví dụ neo theo body thay vì container) có thể khiến layout vỡ trên màn hình nhỏ khi xem qua các thiết bị khác nhau sau khi deploy, vì vị trí tuyệt đối không tự co giãn theo % linh hoạt như flex/grid. Vì vậy nên kết hợp absolute với relative trong phạm vi nhỏ, và ưu tiên Flexbox/Grid cho bố cục tổng thể để đảm bảo website hiển thị đúng trên nhiều thiết bị sau khi deploy lên hosting.

Câu 2 (10đ): Đăng ký domain & cấu hình hosting

Quy trình:

1. Đăng ký domain (ví dụ qua Namecheap, GoDaddy, Mắt Bão...) — chọn tên miền, thanh toán, domain được gắn vào tài khoản quản lý DNS.
2. Chuẩn bị source code website (HTML/CSS/JS, có thể không cần backend nếu là static site).
3. Chọn nơi hosting code (ví dụ GitHub Pages, Netlify, Vercel).
4. Trỏ domain về hosting bằng cách cấu hình bản ghi DNS (A record hoặc CNAME record) tại nơi quản lý domain.
5. Chờ DNS lan truyền (propagate), thường vài phút đến 48 giờ, rồi truy cập domain để kiểm tra.

Vai trò của DNS: DNS (Domain Name System) đóng vai trò "danh bạ điện thoại" của Internet, dịch tên miền dễ nhớ (ví dụ tandev.com) thành địa chỉ IP của server lưu trữ website. Nếu không có DNS, người dùng phải nhớ địa chỉ IP để truy cập trang web.

2 công cụ hỗ trợ deploy miễn phí:

- GitHub Pages: push code lên repo GitHub, bật tính năng Pages trong Settings, GitHub tự build và host static site, có thể gắn domain riêng qua CNAME.
- Netlify: kết nối repo GitHub, Netlify tự động build/deploy mỗi khi có commit mới, hỗ trợ HTTPS miễn phí và quản lý DNS/domain riêng dễ dàng.

Rủi ro bảo mật nếu cấu hình DNS sai: Domain có thể bị "domain takeover" (kẻ xấu chiếm quyền nếu bản ghi CNAME trỏ đến dịch vụ đã bị xoá), website không có HTTPS dẫn đến dữ liệu truyền đi không mã hoá (dễ bị nghe lén/man-in-the-middle), hoặc domain bị downtime nếu cấu hình bản ghi A/CNAME sai địa chỉ.

Câu 3 (10đ): let, const, var — scope & hoisting

var: có function scope (chỉ giới hạn trong hàm chứa nó, không theo block {}), được hoisting lên đầu scope và khởi tạo giá trị undefined ngay từ đầu — nên có thể gọi trước khi khai báo mà không lỗi (nhưng giá trị là undefined).

let: có block scope (giới hạn trong cặp {} chứa nó), cũng được hoisting nhưng nằm trong "temporal dead zone" — nếu gọi trước khi khai báo sẽ bị lỗi ReferenceError.

const: giống let về scope và hoisting, nhưng bắt buộc phải gán giá trị ngay khi khai báo và không thể gán lại (re-assign) tên biến đó sau này (tuy nhiên nếu là object/array thì nội dung bên trong vẫn có thể thay đổi).

Ví dụ lỗi thường gặp:

for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// In ra: 3, 3, 3 (vì var không có block scope,
// cả 3 callback dùng chung 1 biến i sau khi vòng lặp kết thúc)

for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 100);
}
// In ra: 0, 1, 2 (vì let tạo block scope riêng cho mỗi lần lặp)

Cách debug bằng Chrome DevTools: Mở DevTools (F12) → tab Sources, đặt breakpoint tại dòng setTimeout, dùng tab Console để kiểm tra giá trị biến tại từng thời điểm, hoặc dùng console.log/debugger; trong code để theo dõi giá trị biến qua từng vòng lặp, giúp phát hiện vấn đề scope của var so với let.

Câu 4 (10đ): Event Delegation

Event Delegation là kỹ thuật gắn một sự kiện (event listener) duy nhất lên phần tử cha, thay vì gắn riêng lẻ cho từng phần tử con. Khi sự kiện xảy ra trên phần tử con, nó sẽ "nổi bọt" (bubble up) lên phần tử cha, và ta dùng event.target để xác định chính xác phần tử con nào đã được tương tác.

Ví dụ ứng dụng với danh sách động: Với một danh sách sản phẩm được thêm/xoá liên tục bằng JavaScript, thay vì gắn sự kiện click cho từng <li> (sẽ mất tác dụng với phần tử mới thêm sau), ta gắn một listener duy nhất lên <ul> cha:

document.querySelector("ul").addEventListener("click", function(e) {
  if (e.target.tagName === "LI") {
    console.log("Đã click:", e.target.textContent);
  }
});

Lợi ích khi tích hợp với HTML/CSS để tối ưu hiệu suất: Giảm số lượng event listener cần tạo (tiết kiệm bộ nhớ), tự động hoạt động với các phần tử được thêm động sau này mà không cần gắn lại sự kiện, và giúp trang load nhanh hơn khi danh sách có nhiều phần tử (HTML nhẹ hơn về mặt xử lý JS), từ đó tối ưu hiệu suất tổng thể của website.
